# VacancyBot

Асинхронный Telegram-бот для автоматизации поиска вакансий на платформе HeadHunter и оперативного отклика. Проект разработан с целью оптимизации процесса трудоустройства в IT-сфере.

## Основные возможности

- **Умный мониторинг:** Автоматический сбор свежих вакансий по заданным ключевым словам (например, _FastAPI_, _Python Developer_) через симуляцию браузера.
    
- **Фильтрация дубликатов:** Все вакансии сохраняются в базу данных, что гарантирует защиту от повторных уведомлений.
    
- **Удобный интерфейс в Telegram:** Получение карточек вакансий с ключевой информацией (название, компания, ссылка) и интерактивными кнопками управления.
    
- **Быстрый отклик в 1 клик:** Возможность мгновенно отправить отклик на понравившуюся вакансию прямо из чата Telegram, используя официальную интеграцию OAuth.
    
- **Запуск по расписанию:** Фоновый планировщик задач, проверяющий обновления рынка труда каждые 30 минут.
    

## Технологический стек

Проект построен на базе современных асинхронных библиотек и production-ready инструментов:

- **Язык программирования:** Python 3.11+
- **Telegram Bot Framework:** `aiogram 3.x` (асинхронный инструмент нового поколения)
- **Автоматизация браузера:** `Playwright` (асинхронный движок Chromium для парсинга динамического контента и обхода базовой защиты)
- **База данных:** `PostgreSQL`
- **ORM:** `SQLAlchemy 2.0` (с декларативным стилем описания моделей)
- **Миграции БД:** `Alembic`
- **Планировщик задач:** `APScheduler` / `Rocketry`

## Архитектура проекта

Проект разделен на слабосвязанные модули (Loose Coupling), что обеспечивает отказоустойчивость:

1. **Parser Component:** Скрипт на базе `Playwright`, который маскируется под реального пользователя, обходит защиту, собирает HTML-код страниц HeadHunter и извлекает данные с помощью `BeautifulSoup4`.
2. **Database:** Центральное хранилище состояний вакансий и флагов отправки (`is_sent`, `is_applied`).
3. **Telegram Bot Application:** Интерфейс взаимодействия с пользователем и обработки нажатий на инлайн-кнопки.

## Важно: Обход капчи и авторизация (Headless mode)

Поскольку парсинг и отклики происходят через автоматизированный браузер, HeadHunter может потребовать прохождение капчи или авторизацию. Для работы парсера на удаленном сервере в фоновом режиме (`headless=True`) используется стратегия **сохранения профиля (Persistent Context)**.

### Инструкция по подготовке к деплою:

1. **Локальный запуск (с экраном):**
    
    В коде парсера временно установите флаг `is_headless = False` и запустите проект локально на своем компьютере:
    
    Bash
    
    ```
    python parser.py
    ```
    
2. **Прохождение проверки:**
    
    В открывшемся окне браузера Chromium вручную пройдите авторизацию на HH.ru и разгадайте капчу, если она появится.
    
3. **Сохранение сессии:**
    
    После успешного входа в корне проекта автоматически создастся директория `user_data/`. В ней сохранятся все куки, сессии и закэшированные данные вашего профиля.
    
4. **Деплой на сервер:**
    
    Перед отправкой на сервер верните `is_headless = True`. Скопируйте папку `user_data/` на сервер вместе с остальным кодом проекта.
    

> **Как это работает:** При запуске на сервере Playwright подхватит сохраненную сессию из папки `user_data`. HeadHunter посчитает это устройство «доверенным» и пропустит парсер без повторного требования капчи.
